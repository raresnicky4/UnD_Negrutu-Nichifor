1. Introducere

UnD (Unemployment Data Visualizer) este o aplicație Web dezvoltată în cadrul disciplinei Tehnologii Web, cu scopul de a oferi un instrument interactiv de vizualizare și analiză a datelor publice despre șomajul din România.

Datele sunt importate automat de pe portalul open data al Guvernului României (data.gov.ro) și acoperă o perioadă de 29 de luni (ianuarie 2023 – mai 2025), depășind cerința minimă de 16 luni.

Aplicația permite compararea multi-criterială a statisticilor pe județe, nivel de educație, grupe de vârstă, mediu urban/rural, sex și perioadă de timp, cu posibilitatea exportului în formatele CSV, JSON, SVG și PDF.

2. Specificarea Cerințelor

2.1 Cerințe Funcționale

RF-01 – Import date automat

Sistemul importă automat datele despre șomaj de pe data.gov.ro pentru fiecare lună din intervalul selectat, dacă acestea nu există deja în baza de date.

RF-02 – Filtrare multi-criterială

Utilizatorul poate filtra datele după: județ, interval de timp (lună/an start – lună/an stop), mediu (urban/rural), grupă de vârstă, nivel de educație și sex.

RF-03 – Vizualizări grafice

Sistemul oferă minimum 5 tipuri de vizualizări: grafic bare pe județe, grafic tort urban vs. rural, grafic bare pe grupe de vârstă, grafic bare pe nivel de educație și grafic de comparație între județe selectate.

RF-04 – Hartă interactivă

Datele sunt vizualizate cartografic pe o hartă interactivă a României, cu markeri pe fiecare județ și popup cu numărul de șomeri.

RF-05 – Export date

Statisticile și vizualizările pot fi exportate în formatele CSV, JSON, SVG și PDF.

RF-06 – Modul administrare

Există un modul de administrare protejat cu parolă care permite: vizualizarea statisticilor generale, ștergerea cache-ului și reimportul datelor pentru o lună specifică.

RF-07 – Caching

Sistemul implementează un mecanism propriu de caching bazat pe fișiere pentru optimizarea performanței interogărilor frecvente.

2.3 Interacțiunea cu Utilizatorul

Utilizatorul interacționează cu aplicația prin:

  • Selectarea filtrelor din bara de filtrare (județ, perioadă, mediu, vârstă, educație, sex)
  • Apăsarea butonului „Aplică Filtrele" care declanșează un apel Ajax asincron
  • Comutarea între tipurile de grafice prin butoanele de navigare
  • Interacțiunea cu harta (zoom, click pe markeri pentru detalii)
  • Adăugarea județelor în graficul de comparație
  • Exportul datelor prin butoanele CSV, JSON, SVG, PDF
  • Accesarea panoului de administrare prin butonul Admin din colțul paginii

3. Arhitectura Aplicației

3.1 Structura generală

Aplicația urmează o arhitectură client-server bazată pe servicii Web REST. Backend-ul este implementat în PHP fără framework, iar comunicarea cu clientul se face exclusiv prin apeluri Ajax asincrone.

3.2 Structura directorului

core/Security.phpValidare și sanitizare input

3.3 Fluxul de date

  1. Utilizatorul selectează filtrele și apasă „Aplică Filtrele"
  2. app.js construiește URL-ul și face un apel fetch() asincron
  3. api/statistici.php validează parametrii prin Security::validateFilters()
  4. DataImporter verifică și importă datele lipsă de pe data.gov.ro
  5. StatisticiModel execută interogarea SQL cu prepared statements
  6. Rezultatul este stocat în cache și returnat ca JSON
  7. app.js actualizează toate graficele și harta cu datele primite

4. Structura și Proveniența Datelor

4.1 Sursa datelor

Datele provin de pe portalul open data al Guvernului României: data.gov.ro/dataset?q=somaj. Pentru fiecare lună sunt disponibile 4 fișiere CSV:

  • rata – numărul total de șomeri, femei, bărbați și rata șomajului pe județe
  • medii – distribuția urban/rural
  • varste – distribuția pe grupe de vârstă
  • nivel-educatie – distribuția pe nivel de educație

4.2 Structura bazei de date

Baza de date MySQL conține tabela principală statistici cu următoarele coloane:

5. Interfața Web

5.1 Principii de design

Interfața respectă principiile designului Web responsiv, folosind CSS Flexbox și Grid cu media queries pentru adaptarea la diferite dimensiuni de ecran. Paleta de culori este bazată pe albastru (#1e40af) pentru consistență vizuală.

5.2 Componente principale

  • Bara de filtrare – conține 7 filtre: județ, interval de timp, mediu, grupă vârstă, nivel educație, sex
  • Harta interactivă – Leaflet.js cu tile-uri OpenStreetMap, markeri pe toate județele
  • Grafice – 5 tipuri realizate cu Chart.js: bare județe, tort urban/rural, bare vârste, bare educație, comparație județe
  • Butoane export – CSV, JSON, SVG, PDF
  • Buton Admin – acces la panoul de administrare

5.3 Motivații design

S-a ales un design simplu și clar, cu accent pe lizibilitate și funcționalitate. Layout-ul split (hartă stânga, grafice dreapta) permite utilizatorului să vadă simultan distribuția geografică și statisticile numerice. Butoanele rotunjite pentru navigarea graficelor oferă o experiență de utilizare intuitivă.

6. Securitate

6.1 Protecție SQL Injection

Toate interogările SQL folosesc prepared statements cu parametri legați prin PDO. Niciun input al utilizatorului nu este interpolat direct în SQL.

6.2 Protecție XSS

Toate datele afișate în HTML sunt sanitizate cu htmlspecialchars() cu flagul ENT_QUOTES. Inputurile sunt curățate cu strip_tags() și trim() prin clasa Security.

6.3 Validarea inputurilor

Clasa Security::validateFilters() validează toți parametrii primiți: județele sunt verificate față de o listă albă, mediul și sexul sunt validate față de valori permise, iar anii și lunile sunt validate ca întregi în intervale corecte.

6.4 Autentificare admin

Panoul de administrare este protejat prin verificare parolă la fiecare acces POST, fără sesiuni persistente, ceea ce elimină riscul de session hijacking.

8. Concluzie

Aplicația UnD implementează toate cerințele obligatorii ale temei de proiect: vizualizare multi-criterială a datelor despre șomaj pe peste 16 luni, minim 3 tipuri de grafice plus vizualizare cartografică, export în formatele CSV, SVG și PDF, modul de administrare, arhitectură REST cu Ajax asincron, protecție XSS și SQL Injection, design responsiv și caching propriu.

Suplimentar față de cerințele minime, aplicația oferă export JSON, filtru după sex și nivel de educație, grafic de comparație multi-județ cu criterii multiple și un mecanism robust de import automat al datelor cu detectare automată a delimiter-ului CSV și normalizare a numelor de județe.
